  <action>
    **ResolveCartToken middleware** (app/Presentation/Http/Middleware/ResolveCartToken.php):
    Reads `X-Cart-Token` header from request. If missing, returns JSON error `{ success: false, code: 'CART_TOKEN_REQUIRED', message: 'Thieu X-Cart-Token header.', errors: {} }` with status 401. If token provided but cart not found (via CartRepositoryInterface::findByToken), returns `{ success: false, code: 'CART_NOT_FOUND', message: 'Gio hang khong ton tai hoac da het han.', errors: {} }` with status 404. If found, sets `$request->attributes->set('cart', $cart)` where `$cart` is the domain Cart entity, then passes to next middleware. Injects `CartRepositoryInterface` via constructor.

    **CreateCartAction** (app/Application/Order/Actions/CreateCartAction.php):
    Injects `CartRepositoryInterface`. Method `handle(): Cart` calls `$this->carts->create()` which generates UUID and sets expires_at to 7 days from now. Returns domain Cart.

    **AddToCartAction** (app/Application/Order/Actions/AddToCartAction.php):
    Injects `CartRepositoryInterface` and `ProductRepositoryInterface`. Method `handle(int $cartId, int $productId, string $quantity): CartItem`.
    1. Find product via `$this->products->findById($productId)` — throw ProductNotFoundException if null
    2. Check `$product->isActive` — throw InactiveProductInCartException if false (D-06: reject inactive at add time too)
    3. Check if item already in cart via `$this->carts->findItemByCartAndProduct($cartId, $productId)`
    4. If exists: new quantity = `bcadd($existing->quantity, $quantity, 3)` (per D-04: accumulate). Call `$this->carts->updateItemQuantity($existing->id, $newQuantity)`
    5. If not: call `$this->carts->addItem($cartId, $productId, $quantity)`
    6. Refresh cart expiry via `$this->carts->refreshExpiry($cartId)`
    Note: Does NOT check stock (per D-03, D-07, CART-04). Does NOT store price (per D-05).

    **UpdateCartItemAction** (app/Application/Order/Actions/UpdateCartItemAction.php):
    Injects `CartRepositoryInterface`. Method `handle(int $cartItemId, string $quantity): CartItem`.
    Calls `$this->carts->updateItemQuantity($cartItemId, $quantity)`. Quantity must be > 0 (validated in request).

    **RemoveCartItemAction** (app/Application/Order/Actions/RemoveCartItemAction.php):
    Injects `CartRepositoryInterface`. Method `handle(int $cartItemId): void`.
    Calls `$this->carts->removeItem($cartItemId)`.

    **CartResource** (app/Presentation/Http/Resources/CartResource.php):
    Wraps a domain Cart entity. Returns: `id`, `token`, `expires_at`, `items` (collection of CartItemResource), `total_amount` (computed: sum of each item's quantity * product current price, rounded to integer VND). To get current prices, load products from DB in the resource — inject ProductRepositoryInterface or use a computed `$cart->items` where each item has product relation loaded. The resource must query current product prices because D-05 says cart does not lock prices.

    Actually, simpler approach: CartController passes additional product data to the resource. The `show` method in CartController fetches the cart (domain entity) then also fetches all referenced products to get current prices. Pass both to CartResource via `additional()` or a custom wrapper.

    Better approach: Make CartResource work with the Eloquent CartModel directly (like existing ProductResource pattern), so it can access `$cartItem->product->price_vnd` relationship. The controller calls `findByToken` which returns a domain Cart, but for the response we need product prices. So the controller should:
    1. Get cart from middleware: `$cart = $request->attributes->get('cart')` (domain Cart)
    2. Load the Eloquent CartModel with items.product eager loaded for the resource
    3. Return CartResource wrapping the Eloquent model

    OR: Add a method to CartRepositoryInterface that returns the Eloquent model. But that breaks Clean Architecture.

    Best approach: CartResource accepts the domain Cart and an array of products (keyed by id). CartController fetches products for all items in the cart. CartResource computes prices from the products array.

    ```php
    // In CartController::show()
    $cart = $request->attributes->get('cart'); // domain Cart from middleware
    $productIds = array_map(fn($item) => $item->productId, $cart->items);
    $products = []; // fetch from ProductRepositoryInterface by IDs
    return CartResource::make($cart)->additional(['products' => $products]);
    ```

    Actually, simplest and cleanest: Add `findByIds(array $ids): array` to ProductRepositoryInterface if not present, OR just query directly in the resource. Follow existing codebase pattern — check how ProductResource works. Most likely CartResource should work with the Eloquent model for simplicity, following existing patterns. Read the existing resources to decide.

    Final decision for executor: Read existing resource patterns. If resources wrap Eloquent models, do the same for CartResource — have the controller load the Eloquent CartModel with `items.product` eager loaded and pass it to CartResource. The middleware resolves the domain Cart for business logic; the controller can additionally load the Eloquent model for presentation. This is consistent with Clean Architecture: presentation layer can use Eloquent models.
