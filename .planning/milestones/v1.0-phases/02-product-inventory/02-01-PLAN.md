Migration 2 — `create_products_table`:
- `id` bigIncrements
- `category_id` unsignedBigInteger, foreign references `id` on `categories` with `onDelete('restrict')` (per D-07)
- `name` string(255)
- `description` text nullable
- `price_vnd` unsignedInteger (whole VND, no decimals needed for currency)
- `unit_type` string(10) — stores 'con' or 'kg' (per D-02, PROD-02)

- `is_active` boolean default true (per PROD-04)
- `timestamps`

Migration 3 — `create_product_images_table` (per D-02, D-03):
- `id` bigIncrements
- `product_id` unsignedBigInteger, foreign references `id` on `products` with `onDelete('cascade')`
- `path` string(500)
- `thumbnail_path` string(500)
- `is_primary` boolean default false
- `sort_order` integer default 0
- `timestamps`

Migration 4 — `create_stock_adjustments_table` (per D-08, D-09, D-10, D-11):
- `id` bigIncrements
- `product_id` unsignedBigInteger, foreign references `id` on `products` with `onDelete('cascade')`
- `admin_user_id` unsignedBigInteger, foreign references `id` on `users` with `onDelete('restrict')`
- `delta` decimal(10, 3) — can be negative (per D-08)

- `created_at` timestamp (no `updated_at` — adjustments are immutable)
