**Step 1: Application Actions** (follow CreateCategoryAction pattern)

`app/Application/Product/Actions/CreateProductAction.php`:
- Constructor inject `ProductRepositoryInterface`, `CategoryRepositoryInterface`
- `handle(string $name, ?string $description, int $priceVnd, string $unitType, int $categoryId, string $stockQuantity, bool $isActive): Product`
- Validate category exists via `$this->categories->findById($categoryId)` — throw `CategoryNotFoundException` if null
- Validate `$unitType` matches UnitType enum values ('con' or 'kg')
- Call `$this->products->create([...])` with all fields
- Return Product entity

`app/Application/Product/Actions/UpdateProductAction.php`:
- Constructor inject `ProductRepositoryInterface`, `CategoryRepositoryInterface`
- `handle(int $id, string $name, ?string $description, int $priceVnd, string $unitType, int $categoryId, bool $isActive): Product`
- Load existing product; throw `ProductNotFoundException` if not found
- Validate category exists; throw `CategoryNotFoundException` if null
- Call `$this->products->update($id, [...])` — do NOT update stock_quantity here (stock managed separately per INVT-02)
- Return updated Product entity

`app/Application/Product/Actions/DeleteProductAction.php`:
- Constructor inject `ProductRepositoryInterface`
- `handle(int $id): void`
- Load existing product; throw `ProductNotFoundException` if not found
- Call `$this->products->delete($id)`

`app/Application/Product/Actions/ToggleProductActiveAction.php` (per PROD-04):
- Constructor inject `ProductRepositoryInterface`
- `handle(int $id): Product`
- Load existing product; throw `ProductNotFoundException` if not found
- Toggle: call `$this->products->update($id, ['is_active' => !$product->isActive])`
- Return updated Product entity
