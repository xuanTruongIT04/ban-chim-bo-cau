    **UpdateOrderStatusAction** (app/Application/Order/Actions/UpdateOrderStatusAction.php):
    Injects `OrderRepositoryInterface`.
    Method `handle(int $orderId, OrderStatus $newStatus): Order`:
    1. Find order via `$this->orders->findById($orderId)` — throw OrderNotFoundException if null
    2. Check `$order->orderStatus->canTransitionTo($newStatus)` — throw InvalidOrderTransitionException(from, to) if false (per D-12, ORDR-07)
    3. Call `$this->orders->updateStatus($orderId, $newStatus)` — return updated Order

    This handles forward transitions AND 1-step back transitions (D-11) because the `allowedNextStates()` on the enum already includes them. It does NOT handle cancellation (that's CancelOrderAction which needs stock restoration).

    **CancelOrderAction** (app/Application/Order/Actions/CancelOrderAction.php):
    Injects `OrderRepositoryInterface`, `ProductRepositoryInterface`.
    Method `handle(int $orderId): Order`:

    Per D-09: only admin calls this. Per D-10: allowed at any state except hoan_thanh. Per ORDR-05: stock restored in same transaction.

    ```php
    return DB::transaction(function () use ($orderId): Order {
        $order = $this->orders->findById($orderId);
        if ($order === null) {
            throw new OrderNotFoundException($orderId);
        }

        if (!$order->orderStatus->isCancellable()) {
            throw new InvalidOrderTransitionException(
                $order->orderStatus,
                OrderStatus::Huy
            );
        }

        // Restore stock for each order item — lock product rows to prevent race
        $sortedItems = collect($order->items)
            ->sortBy('productId')
            ->values();

        foreach ($sortedItems as $item) {
            $product = $this->products->findByIdForUpdate($item->productId);
            if ($product !== null) {
                $restoredStock = bcadd($product->stockQuantity, $item->quantity, 3);
                $this->products->updateStock($item->productId, $restoredStock);
            }
        }

        return $this->orders->updateStatus($orderId, OrderStatus::Huy);
    });
    ```

    Note: sorts items by productId ascending to prevent deadlocks (same pattern as PlaceOrderAction). Uses bcadd to add quantity back. Product might have been deleted — if findByIdForUpdate returns null, skip (defensive).

    **ConfirmPaymentAction** (app/Application/Order/Actions/ConfirmPaymentAction.php):
    Injects `OrderRepositoryInterface`.
    Method `handle(int $orderId): Order`:
    1. Find order — throw OrderNotFoundException if null
    2. Check order is not already `da_thanh_toan` — if so, return order as-is (idempotent)
    3. Check order is not cancelled (`order->orderStatus !== OrderStatus::Huy`) — throw exception if cancelled
    4. Call `$this->orders->updatePaymentStatus($orderId, PaymentStatus::DaThanhToan)` (per PAYM-04)
